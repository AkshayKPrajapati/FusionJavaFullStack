Question 1: What is JDBC and its benefits?
Answer: JDBC (Java Database Connectivity) is an API in Java that provides methods for executing SQL statements and accessing any tabular data as a database. It offers a set of classes and interfaces that enable Java applications to interact with databases.

Benefits of JDBC:
1. Database Independence: Write once, run anywhere
2. Type Safety: Compile-time checking
3. Performance: Direct SQL execution
4. Rich API: Comprehensive database operations
5. Connection Pooling: Efficient resource management

Question 2: Why do we need JDBC?
Answer: JDBC is needed to:
1. Connect to databases
2. Execute SQL queries
3. Process results
4. Handle transactions
5. Manage connections efficiently


Question 3: Why we store in database?
Answer: We store data in databases because:
1. Persistence: Data survives application restarts
2. Data Integrity: Ensures accuracy and consistency
3. Concurrent Access: Multiple users can access simultaneously
4. Security: Controlled access and authentication
5. Scalability: Handle large amounts of data
6. Querying: Efficient data retrieval and manipulation
7. Backup and Recovery: Data protection mechanisms

Question 4: Why we not store in file?
Answer: We don't store data in files because:
1. No concurrent access
2. No data integrity
3. No security
4. No scalability
5. No efficient querying
6. No backup and recovery
7. No transaction support

Question 5: What is JDBC API?
Answer: JDBC API is a set of classes and interfaces that provide a standard way to access databases from Java applications. It consists of two layers:
1. JDBC API: Provides a set of classes and interfaces for database connectivity
2. JDBC Driver API: Provides a set of classes and interfaces for database drivers


Question 6: What are the different types of JDBC drivers?
Answer: There are four types of JDBC drivers:
1. Type 1: JDBC-ODBC Bridge Driver -> Bridge between JDBC and ODBC
2. Type 2: Native API Driver -> Native API for database access
3. Type 3: Network Protocol Driver -> Network protocol for database access
4. Type 4: Pure Java Driver -> Pure Java for database access -> every  database wander design  

Question 7: what is driver?
Answer: A driver is a software component that enables communication between a database and an application. It translates SQL statements into database-specific commands.

Question 8: what is driver manager?
Answer: A driver manager is a class that manages JDBC drivers. It is responsible for loading and registering drivers, and creating connections to databases.

Question 9: what is drawback of JDBC-ODBC bridge driver?
Answer: there are 2 main drawbacks:
1. Platform dependent - only works on Windows operating system
2. Performance overhead - adds an extra layer (ODBC) between JDBC and database, resulting in slower performance compared to pure JDBC drivers

Question 13:Pure Java Driver or Thin Driver?
Answer : Pure Java Driver or Thin Driver is a type of JDBC driver that is written entirely in Java and does not require any native code or third-party software to connect to the database.There are no drawback yet.

package demo.example;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;

public class GetEmpolyeeRecoards {
	public static void main(String [] args) {
		Connection connection=null;
		try {
			//step 1.load Driver
			Class.forName("com.mysql.cj.jdbc.Driver");
			System.out.println("Driver class loaded!!");
			
			//step 2.Connection Establish to the database
			String url="jdbc:mysql://localhost:3306/016_25batch";
			String username="root";
			String password="local";
			
			connection = DriverManager.getConnection(url, username, password);
			System.out.println("Connection establish successfully");
			
			//step 3. write a SQL Query
			String sql = "select * from employee";
			
			//step 4.Associate Query with connection
			Statement statement = connection.createStatement();
			
			//step 5.Execute Query
			ResultSet resultSet = statement.executeQuery(sql);
			
			//step 6. Iteration the data
			while(resultSet.next()) {
				System.out.println("Id: "+resultSet.getInt(1));//column Index
				System.out.println("Name : "+resultSet.getString(2));//column Index
				System.out.println("Salary : "+resultSet.getFloat("salary"));//Column Name
				
				
				System.out.println("--------------");
				
			}
		} catch (ClassNotFoundException | SQLException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
		//step 7 .close the connection
		finally
		{
			try
			{
				if(connection!=null)
				{
					connection.close();
				}
			}
			catch (SQLException e)
			{
					// TODO Auto-generated catch block
					e.printStackTrace();
			}
		}
	}
}

Question 14: What is the difference between execute() and executeQuery() methods in Statement class?
Answer : 

The execute() : this method is used to execute any SQL statement,
return boolean value

The executeQuery() : this method is used to execute SELECT statements only. return ResultSet object;
ResultSet resultSet = statement.executeQuery(sql);

The executeUpdate() : this method is used to execute INSERT, UPDATE, DELETE statements. return int value;
int rowCount = statement.executeUpdate(sql);
